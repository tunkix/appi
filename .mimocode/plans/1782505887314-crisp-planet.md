# Fix: email missing from /users API response

## Root cause

`$user->email = $user->getEmail()` in `UserService.php:24,36` triggers `Entity::__set()` which calls Shield's `setEmail()` and returns early — writing to a private property, never to `$this->attributes`. Since `toArray()`/`jsonSerialize()` only iterates `$this->attributes`, email is silently dropped from JSON output.

## Changes

### 1. `app/Entities/UserEntity.php` — extend Shield's User, override jsonSerialize()

Replace the current class (which extends base `Entity` and is unused) with:

```php
final class UserEntity extends \CodeIgniter\Shield\Entities\User
{
    protected $casts = [
        'id'         => 'integer',
        'created_at' => 'datetime',
        'updated_at' => 'datetime',
        'deleted_at' => '?datetime',
    ];

    public function jsonSerialize(): array
    {
        $data = parent::jsonSerialize();
        $data['email'] = $this->getEmail();

        return $data;
    }
}
```

This follows CI4's entity pattern — `jsonSerialize()` is the right hook since the controller uses `$this->respond()` which JSON-encodes entities.

### 2. `app/Models/Core/UserModel.php` — set returnType to UserEntity

Add:

```php
protected $returnType = \App\Entities\UserEntity::class;
```

### 3. `app/Services/UserService.php` — remove dead email lines

Delete lines 24 and 36 (`$user->email = $user->getEmail()`). The `jsonSerialize()` override now handles email automatically. Keep `withIdentities()` calls (needed by `getEmail()` to lazy-load from `auth_identities` table) and `groups` assignment (works because no `setGroups()` setter exists).

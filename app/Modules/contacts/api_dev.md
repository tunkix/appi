# Contacts Module – API Documentation

## 1. Overview
Provides RESTful endpoints for managing **Contacts**, including custom fields, company information, job titles, and contact methods (email, phone, etc.).

## 2. Authentication & Permissions
All routes are protected by Shield's JWT/Token authenticator filter. Every API call must include a valid Bearer Token in the `Authorization` header.
Required permissions (checked inside each method or via filter):
- `contacts.view` – list or view contacts
- `contacts.create` – create a new contact
- `contacts.edit` – edit an existing contact
- `contacts.delete` – delete a contact
- `contacts.manage` – full access (view/create/edit/delete/export)

## 3. Base Route
`/api/contacts`

## 4. Endpoints
| Method | Path | Description | Required Filters / Body |
|--------|------|-------------|--------------------------|
| **GET** | `/api/contacts` | List contacts. Supports filters defined in `api_schema`. | `search` (string), `company` (string), `job_title` (string), `date_from`/`created_after` (date), `date_to`/`created_before` (date) |
| **GET** | `/api/contacts/(:num)` | Retrieve a single contact by its ID. | – |
| **POST** | `/api/contacts` | Create a new contact. Handles custom fields automatically. | JSON body with **fixed_fields** (`first_name`, `last_name`, `email`, `company`, `job_title`, `notes`) plus any custom field keys defined in the module's `module.json` (`custom_fields` is generated by the service). |
| **PUT/PATCH** | `/api/contacts/(:num)` | Update an existing contact. | Same JSON body as create. |
| **DELETE** | `/api/contacts/(:num)` | Delete a contact. | – |

### Sample Requests
```bash
# List contacts filtered by company
curl -X GET "http://localhost:8080/api/contacts?company=Acme" \
  -H "Authorization: Bearer <TOKEN>" -H "Accept: application/json"
```
```bash
# Create a new contact with a custom field "favorite_color"
curl -X POST "http://localhost:8080/api/contacts" \
  -H "Authorization: Bearer <TOKEN>" -H "Accept: application/json" -H "Content-Type: application/json" \
  -d '{"first_name":"Jane","last_name":"Doe","company":"Acme","job_title":"Manager","notes":"Important client","favorite_color":"blue"}'
```

### Response Format
Each contact record returns all fixed fields plus decoded `custom_fields` (object) and a `methods` array with each entry containing `type`, `value`, and `label`.

## 5. SmartPicker Compatibility
Fields exposed via `shareable_fields` can be used with `/api/search/picker`:
- `first_name`
- `last_name`
- `company`
- `job_title`
- `notes`

## 6. Error Handling
Follows the global format (401, 403, 404, 422, 500) described in the main API guide.
